## Overview

Advanced puzzles demonstrate Kamotake's SAT-based solver handling larger grids, complex constraint patterns, and the computational challenges of difficult nonograms.

## Performance and Complexity

As puzzle size increases, the number of CNF clauses grows significantly:

- **Variables**: Approximately `ROWS × COLUMNS` boolean variables
- **Clauses**: Depends on constraint complexity, can reach thousands or millions
- **Solve time**: Ranges from milliseconds to seconds (or more for very hard puzzles)

The `-d` flag preserves intermediate files, letting you inspect the generated CNF and SAT solver output.

### Characteristics

- **Grid size**: 25 rows × 25 columns = 625 cells
- **Full rows/columns**: Several constraints equal to grid size (25)
- **Empty constraint**: Row 8 has `0` (completely empty)
- **Large blocks**: Constraint `4 17` means 4 filled, gap, then 17 filled

### Analysis

This puzzle showcases interesting SAT encoding challenges:

1. **Full rows**: Constraint `25` means all cells filled → simplified clauses
2. **Empty rows**: Constraint `0` means all cells empty → inverse clauses
3. **Large blocks**: `4 17` in a 25-wide row leaves only 3 possible positions

- **Sys**: System call overhead

## Understanding CNF Generation

With the `-d` flag, you can examine the CNF file:

```bash
./kamotake -d data/test7.txt
head -20 *.cnf  # View first 20 lines of generated CNF
```

The CNF file uses DIMACS format:
```
p cnf VARIABLES CLAUSES
1 -2 3 0
-1 2 0
...
```

Each line is a clause (disjunction of literals), terminated by `0`.

## Performance Optimization Tips

1. **Dense puzzles** (many filled cells): Usually faster due to constraint propagation
2. **Sparse puzzles** (few filled cells): May require more backtracking
3. **Many small blocks**: More complex encoding, more clauses
4. **Few large blocks**: Simpler encoding, faster solving

## Troubleshooting

If a puzzle takes too long:

1. **Check the CNF size**: `wc -l *.cnf` after running with `-d`
2. **Verify the puzzle**: Ensure constraints are solvable
3. **Try a different SAT solver**: Kamotake uses MiniSat; other solvers may perform differently
